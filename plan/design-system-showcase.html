Foundations

Color, tokens, type, rhythm

Every value is a token. The whole system flips between light and dark from one set of custom properties, and the same set is the single surface an adopter overrides to re-skin the portal. Contrast is verified, not assumed: WCAG 2.2 AA is the floor.

Palette and semantic color

Key text and surface pairs are annotated with their measured contrast ratio. Semantic hues (success, warning, critical) sit apart from the brand accent and never carry state alone: they always travel with text or an icon.

Surface 02 / Admin

Form builder: QCMS's own identity

The authoring surface is not re-skinned: it wears QCMS's Cobalt identity and is denser, more opinionated, and data-forward. Technical content is first-class, set in the monospace face. Question versions are pinned and explicit, conditions are shown as the rule that actually runs, and validation is live. Light and dark.

QCMS Design System / first visual pass / v0.1. Every color pair on this page meets WCAG 2.2 AA in both themes (4.5:1 body text, 3:1 large and UI). Fonts are system stacks: no external requests. Components are static representations of the a2-react-aria / shadcn vocabulary: table, dialog, radio, text-field, number-field, tag, alert, button, card, breadcrumb, menu.
